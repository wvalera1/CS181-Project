Exploring Cryptocurrency Market Trends Using CoinLore and CoinGecko API Data

In this project, we will be utilizing the CoinLore and CoinGecko APIs to retrieve data for real-time and historical cryptocurrency data, exploring market trends of various cryptocurrencies.

Introduction

Cryptocurrencies have revolutionized the world of finance, offering decentralized and secure digital transactions. This project aims to explore market trends in the cryptocurrency world using data from the CoinLore and CoinGecko APIs. CoinLore offers a public and free cryptocurrency API for developers and researchers, providing independent data for over 12,000 crypto coins and more than 300 crypto exchanges. CoinGecko is another public API that offers historical data for over 15,000 coins and 1,000 exchanges. Our aim is to utilize these APIs to understand how different cryptocurrencies perform, particularly Bitcoin (BTC) and Ethereum (ETH), and identify key factors by observing their real-time price, historical trends, and current market data. When collecting and analyzing data, it is important to consider the ethical implications. The use of public APIs, such as CoinLore and CoinGecko, ensures that the data we use is freely available to the public and that this data is solely intended for research purposes. By using public APIs, we also ensure that we are respecting user privacy and data security, and that we are also not exposing sensitive information. To sum up, this project aims to utilize the CoinLore and CoinGecko API to analyze market trends and performance indicators to obtain a deeper understanding of the key factors that drive the cryptocurrency market.

# Fetch historical data from CoinGecko API
coin_id = 'bitcoin'
api_url = f"https://api.coingecko.com/api/v3/coins/{coin_id}/market_chart?vs_currency=usd&days=7"
response = requests.get(api_url)
data = response.json()

# Convert JSON data to DataFrame
prices = data['prices']
bitcoin_df = pd.DataFrame(prices, columns=['timestamp', 'price_usd'])

# Convert timestamp to datetime
bitcoin_df['timestamp'] = pd.to_datetime(bitcoin_df['timestamp'], unit='ms')

# Create line plot
plt.figure(figsize=(14, 6))
plt.plot(bitcoin_df['timestamp'], bitcoin_df['price_usd'])
plt.title("Price Changes Over Time for Bitcoin in the Past Week")
plt.xlabel('Date')
plt.ylabel('Price (USD)')
plt.xticks(rotation=45)
plt.grid(True)
plt.show()

This graph displays the price changes in Bitcoin in the past week. The graph highlighs bitcoin's volatility, as it has seen price increases and decreases over the past 7 days. The graph indicates that the price of Bitcoin is currently decreasing. The price of Bitcoin recently spiked up to 100,000 USD, peaking at 102,000 USD before dropping. The lowest Bitcoin has been this week is about 93,000 USD, which was on December 10, 2024.

plt.title('24-Hour Trading Volume of Top 20 Cryptocurrencies')

plt.ylabel('Trading Volume (USD)')
plt.xticks(rotation=45)
plt.show()

This graph showcases the 24-hour trading volume of the top 20 cryptocurrencies (by market cap). The trading volume of a cryptocurrency (volume24) is the amount of that cryptocurrency that has been bought and sold between traders in the past 24 hours. Higher trading volumes tend to mean that these cryptocurrencies are the most popular among traders, likely due to their price, volatility, liquidity, and market recognition. Cryptocurrencies with high volatility (such as BTC and ETH) appeal to many traders who are looking to make a quick profit from the rapid price swings. Generally, high trading volumes indicate prices are rising and low trading volumes indicate prices are dropping. However, this doesn't seem to be the case for Tether (USDT). Tether is a stablecoin, which is a type of cryptocurrency that keeps its price stable through various market mechanisms. Tether tends to be popular among investors who keep value in the cryptomarket, but don't want to hassle with the volatility that a typical coin faces. The price of Tether always remains 1 USD, and does not appreciate or depreciate. By providing liquidity and stability, Tether tends to rank among the top cryptocurrencies with the highest trading volume.

In this reflection, we will try to share our coding choices, taking into consideration performance, human readability, code dependencies, and reproducibility.

Performance

We utilized the requests library to fetch data from the CoinGecko and CoinLore APIs. This library allows us to send HTTP requests using Python, which made retrieving data an easy and efficient process. We then converted our responses into JSON data, and utilized pandas to create DataFrames, which are optimized for performance and data manipulation. We also used the pd.to_numeric() function to convert columns of string into numeric types, allowing us to perform operations like sorting and filtering more efficiently. Similarly, we also utilized the pd.to_datetime() function to make graphing historical data an easier process. By using these libraries, we are ensuring that are code is optimized for handling larger datasets.

Human Readability

Our code is organized in a way that flows logically, starting from fetching the data, converting it, filtering it, and then finally displaying/visualziing it. We also made sure to use clear and descriptive variable names such as tickers_df, top_20_df, and most_expensive to make our code easy to understand and improve readability. Lastly, we made sure to add comments and documentation throughout the code that helps explain the purpose of each section and the rationale behind our code, improving clarity and allowing readers to follow our logic.

Code Dependencies

We used requests to efficiently retrieve the data we need, pandas to manipulate and filter our data, and seaborn, matplotlib to create and customized visualizations and plots. These libraries are well-documented, actively maintained, and widely-popular among Python users. By using these popular, up-to-date libraries, we can likely share our code with other users who might be familiar with it, and we also ensure that our code is stable and is at minimal risk of compatibility issues.
